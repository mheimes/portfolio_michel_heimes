UI/UX

UI/ UX Slogans:

  1. You ≠ User
    • You = designers that design a webpage or any interface that should be used by common users
    • You are not the user means that processes and steps that might be obvious to take for designers are not automatically obvious for the standard user.
    • User testings and watching them interact with a prototype/ mock-up/ etc. is an appropriate way to test what is a good and easy-to-understand design for users and what’s not!
  1. Keep it simple
    • The fewer features you have, the easier the product will be for users. (The fewer options you have for a given feature, the easier that feature will be to use.) → Any time you add something, usability goes down because the user interface becomes more complicated
    • Avoid Feataritis and keep the user interface as simple as possible
    • Extra Features: → slow down users → make it harder to use existing features → increase the error rate
    • Usefulness requires both usability and utility → So it must be designed to be used as easy as possible but must also accomplish something people need
  1. You can’t impose joy.
    • You cannot dictate what features/ parts people enjoy.
    • You should avoid implementing features whose only purpose is to look great and impress the user, if the feature delays the usage of the interface. → Examples:
      • Complex animations of buttons when clicked etc. can be exciting once but can get old and annoying when the user has to use it multiple times.
      • Splash pages (the one websites had in the time around the dot com bubble) that appeared before the website’s content and took some time to finish until the user could use the actual functionality of the website. Their only purpose was to “lighten up” the overall user experience with a nice animation or something similar before they enter the rather boring website. Still this feature is just in the way of the user and tries to hide the bad actual website.
  1. Communication not decoration
    • Avoid design for design’s sake when it does not make the communication simpler and cleaner.
    • In a user interface, every additional element is something that the user has to figure out and that detracts from their attention to the actual communicative elements in the design.
    • The user doesn’t know what’s communication and what’s decoration and even later the decoration can divert visual attention.
  1. Brevity = Brilliance (Prägnanz/ Kürze = Brillanz)
    • The more concise your communication the higher the probability that the user will pay attention to what you say
      • The lower the word count, the more the user is likely to understand the content and remember it later.
      • General advice: Cut the word count in half compared to the amount you’d write for print. → People are more motivated to read more in print than on the web since the user has that piece of content in their hand, which is something they maybe even payed good money for and where they want to dig deeper. → On the web there are at least ten other websites one click away, so it takes very little effort to go and read something else than you content if you don’t get to the point quickly.
    • What’s important is not how much information is on the webpage, but how much information is in the user’s head after they have visited your site. → Keeping it clean means that users retain more information, which again mean that they think you are more brilliant, than if you used more words that they just won’t remember.
